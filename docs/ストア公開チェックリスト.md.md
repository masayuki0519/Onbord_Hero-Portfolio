# Store Review Checklist

Consolidated, checkable version of the "ストア審査" notes already sketched
in `README.md` ("ストア申請前に追加するとよいもの") and `ROADMAP.md`
(section 6). Use this as the actual pre-submission checklist; update the
prose docs first if scope changes, then reflect the change here.

## Legal / Policy

- [ ] `PRIVACY.md` is published at a stable public URL (required by both
      Google Play and Apple App Store Connect).
- [ ] Privacy policy URL is entered in the Play Console listing and in
      App Store Connect (if iOS is shipped).
- [ ] Play Console **Data safety** form matches `PRIVACY.md` exactly
      (currently: no data collected/shared, local-only `localStorage`
      save data).
- [ ] If/when a real rewarded-ad SDK (e.g. AdMob) replaces the current
      simulated ad flows (chest-limit boost and ad-open-a-chest — see
      `app.js`, "本番ではAdMobなどの..."),
      **before submitting**: update `PRIVACY.md` to disclose the ad SDK,
      advertising ID usage, and any data it collects, and update the Data
      safety form accordingly. Do not submit with a privacy policy that
      says "no ad SDK" while a real ad SDK is integrated.
- [ ] Age rating questionnaire completed (Google Play IARC / Apple).
      Note in `README.md`: no gambling, no real-money rewards, no external
      trading — keep those claims true before answering the questionnaire.
- [ ] Confirm in-game reward odds (chest drop rates) are still disclosed
      in-game, per the existing `README.md` policy ("無料宝箱の排出率を
      ゲーム内に明記").

## Permissions

- [ ] "他のアプリの上に表示" (draw-over-other-apps, used by the mini
      window overlay) has a clear in-app explanation of why it's needed
      and what happens if the user denies it (per `ROADMAP.md` section 6).
- [ ] No unused/unexplained permissions remain in
      `android/app/src/main/AndroidManifest.xml`.

## Build

- [ ] Release build uses a real signing key (not debug signing).
- [ ] AAB (Android App Bundle) generated for Play Store submission,
      not just a debug/release APK.
- [ ] `targetSdkVersion` / `compileSdkVersion` in
      `android/app/build.gradle` meet the current Play Store minimum
      target API requirement at submission time (this changes yearly —
      check the current requirement on the Play Console before building).
- [ ] Internal testing track → closed testing → production, per
      `ROADMAP.md` section 6.

## Store Listing Assets

- [ ] App icon ≥ 1024×1024 (Play Store listing) plus adaptive icon for
      the app itself.
- [ ] Phone screenshots (minimum required count/aspect ratio per current
      Play Console requirements).
- [ ] Short description / full description drafted (Japanese; add other
      locales only if actually localized).
- [ ] Feature graphic (Play Store banner, 1024×500).

## In-App

- [ ] Title screen, first-launch flow, and returning-save flow all tested
      end-to-end on a real device (see `docs/TODO.md` "Verify Portal UI on
      device").
- [ ] Volume/mute controls present if audio is enabled by default
      (`README.md` note: "音声を入れる場合はミュート設定").
- [ ] In-game "セーブデータ削除" (delete local save) still works and is
      reachable, matching the `PRIVACY.md` claim.

## Post-Submission

- [ ] Keep `PRIVACY.md`, this checklist, and the Data safety form in sync
      whenever a new SDK, permission, or data flow is added — do this in
      the same commit/PR as the feature, not as a follow-up.
